# Экспертная система подбора мест производственной практики на Prolog

Интеллектуальная система, разработанная для студентов техникума (колледжа) с целью автоматизированного подбора наиболее подходящих предприятий для прохождения производственной практики. Система учитывает специальность обучения, средний балл (GPA), географические предпочтения и профессиональные навыки студента, рассчитывая процент соответствия каждой вакансии.

Проект реализован исключительно на логическом языке программирования **Prolog** и предназначен для запуска в стандартной среде **SWI-Prolog**.

---

## 🛠 Структура проекта

1. **`expert_system.pl`** — база знаний, содержащая факты о компаниях-партнерах (`company/6`), должностях практики (`position/6`) и правилах логического вывода для подбора (проверка специальности, GPA, локации и подсчет совпадения навыков).
2. **`run_cli.pl`** — интерактивный консольный интерфейс. Опрашивает пользователя о его профиле и выводит ранжированный список подходящих вакансий с рекомендациями по изучению недостающих навыков.
3. **`test_input.txt`** — файл готовых ответов для автоматического тестирования логического вывода.
4. **`generate_docx_report.py`** — скрипт на Python для генерации файла курсовой работы `Курсовая_работа.docx` по ГОСТу.
5. **`Курсовая_работа.docx`** — готовый отформатированный текстовый документ курсовой работы для сдачи.

## 📋 Описание логики правил Prolog

Основной предикат рекомендации:
```prolog
recommend_position(StudentSpec, StudentSkills, StudentGPA, StudentLoc, 
                   PositionID, Title, CompanyName, CompanyLoc, Score, MissingSkills)
```

### Критерии отбора (жесткие фильтры):
1. **Специальность (`match_specialization/2`)**: Вакансия должна соответствовать специальности обучения студента (`is` — программирование, `set` — администрирование, `tm` — машиностроение, `buh` — учет, `tur` — туризм).
2. **Академический ценз (`match_gpa/2`)**: Средний балл студента должен быть не ниже минимального порога компании (например, ≥ 4.5 для Центробанка, ≥ 3.5 для ООО "ПромТех").
3. **Расположение (`match_location/3`)**: Локация компании должна совпадать с городом студента, либо компания должна предлагать удаленную работу (`remote`), либо студент готов поехать в любой город (`any`).

### Расчет процента соответствия (Score):
* Прохождение жестких фильтров дает **базовые 50% соответствия**.
* Оставшиеся **50% распределяются пропорционально** количеству совпадающих навыков студента и требований вакансии (если у вакансии 3 требования, и у студента есть все 3 — он получает дополнительные 50%, итого 100%).
* Недостающие навыки возвращаются в переменной `MissingSkills` в виде рекомендаций для изучения.

## 🚀 Запуск системы в SWI-Prolog

Для запуска интерактивного опроса выполните команду в терминале (`PowerShell` или `cmd`) в папке проекта:

* Если `swipl` добавлен в системный PATH:
  ```bash
  swipl -q -g start -t halt run_cli.pl
  ```
* Если PATH еще не обновился (запуск по прямому пути Windows):
  ```powershell
  & "C:\Program Files\swipl\bin\swipl.exe" -q -g start -t halt run_cli.pl
  ```

### Правила ввода данных:
При ответе на вопросы в консоли обязательно ставьте точку `.` в конце каждого значения и нажимайте `Enter`:
1. Специальность: `is.` (или `set.`, `tm.`, `buh.`, `tur.`).
2. Средний балл: `4.5.` (число с плавающей точкой).
3. Город: `moscow.` (или `spb.`, `ekaterinburg.`, `any.`).
4. Навыки: Вводите по одному, например `python.`, затем `sql.`, `git.`. Для окончания ввода напишите `done.` и нажмите `Enter`.

## 🧪 Автоматическое тестирование

Для автоматического прогона тестового сценария выполните команду:
```powershell
Get-Content test_input.txt | & "C:\Program Files\swipl\bin\swipl.exe" -q -g start -t halt run_cli.pl
```
Это позволит мгновенно увидеть результаты логического вывода для студента со средним баллом `4.5`, специальностью `is` (ИТ) в Москве, владеющего технологиями `python`, `sql`, `git`.
